# Приложение анкеты

## Общее описание
Приложение является клиентской частью системы управления данными пользователей и анкет. Оно обеспечивает взаимодействие с удаленным сервером через `REST API` для выполнения поиска, фильтрации, удаления анкет и пользователей, добавления данных о прохождении, создании новых анкет, редактировании анкет и профилей.

## Описание архитектуры
Был применен паттерн `MVVM (Model-View-ViewModel)`.

Приложение использует одну Activity (архитектура `Single-Activity`). Вся навигация между экранами реализована через замену `Fragment` с использованием библиотеки `Navigation Component`.

Проект организован по принципу `feature-first`.
В `common` находятся модули, используемые во многих модулях приложения.
`data` включает в себя конфигурацию `Retrofit`, а также `EncryptedSharedPreferences` для безопасного хранения `JWT-токенов`.
В `di (Dependency Injection)` модуль настройки `Koin` — библиотеки для внедрения зависимостей.
В `features` находятся модули для каждой функции приложения. Каждая функциональность имеет свою структуру. Она делится на `ui` и `domain`. `domain` содержит функции для обращения к моделям, получению и отправке данных.
`ui` содержит фрагменты, адаптеры и `ViewModels`.

## Используемые библиотеки
- `Retrofit` - HTTP-клиент для выполнения сетевых запросов и сериализации/десериализации JSON.
- `Gson` - библиотека для сериализации и десериализации JSON.
- `Security Crypto (EncryptedSharedPreferences)` - библиотека для безопасного хранения ключей и небольших объемов конфиденциальных данных в зашифрованном виде.
- `Navigation Component` - фреймворк для управления навигацией в приложениях с одной Activity.
- `Koin` - библиотека для внедрения зависимостей (Dependency Injection) на Kotlin.
